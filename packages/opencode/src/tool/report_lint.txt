Audit a ULMCode operation before report handoff.

Use this before final reporting and after report edits. It checks whether the operation has a checkpoint ledger and whether findings are evidence-backed, validated, and report-ready enough for a defensible client deliverable. For long reports, use requireOutlineBudget=true so the report must satisfy the target_pages budget from reports/report-outline.md; tune minOutlineWordsPerPage when the default 300 words/page is too high or low for the deliverable. Also use requireFindingSections=true and minFindingWords to catch sparse individual finding writeups even when total word count is high. For final handoff, set finalHandoff=true so the operation must be complete, in handoff stage, and have the operation plan, rendered deliverables, and runtime summary.
