Write a durable runtime handoff summary for a ULMCode operation.

Use this during long autonomous runs, before compaction-heavy handoffs, and before final delivery. The summary records model-call split, token/cost budget rollups, compaction pressure, repeated fetches, persisted background task state, stale-task restart args when available, notes, and canonical artifact paths under `.ulmcode/operations/<operation>/deliverables/`.

When model-call, token/cost, compaction, or background task fields are omitted, the tool derives them from the current session's assistant messages, persisted child subagent sessions, persisted background job sessions, and the persisted background job ledger. Provide explicit values only when you need to override or supplement the stored runtime ledger.
