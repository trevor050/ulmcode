Create a compact restart brief for a ULMCode operation from disk state.

Use this before resuming a long-running operation after context compaction, process restart, handoff, or uncertainty about current state. The brief lists checkpoint state, health gaps, active/background tasks, recommended follow-up tools, and a continuation prompt that can restart work without rereading every artifact manually. For unattended runs, pass staleAfterMinutes so stale checkpoints and stale background jobs are called out before new work starts; use `operation_recover` when the brief reports restartable stale lanes.
